## L'écosystème des outils IA

Il sort un nouvel outil IA chaque semaine. C'est impossible de tout suivre, et c'est normal. L'objectif n'est pas de tout connaitre, mais de savoir naviguer dans l'écosystème pour choisir le bon outil au bon moment.

### Les grandes catégories d'outils

L'écosystème IA pour les professionnels tech se découpe en plusieurs familles.

#### Assistants de code

Ces outils s'intègrent directement dans votre éditeur ou votre terminal. Ils complètent du code, génèrent des fonctions, et répondent à des questions sur votre codebase.

| Outil | Type | Points forts |
|-------|------|-------------|
| GitHub Copilot | Extension IDE | Intégration VS Code native, large base d'utilisateurs |
| Cursor | IDE complet | IDE dédié IA, édition multi-fichiers, context awareness |
| Claude Code | CLI | Agentique, accès filesystem, autonomie élevée |
| Cody (Sourcegraph) | Extension IDE | Recherche sémantique sur le codebase |
| Windsurf | IDE complet | Flows agentiques, bonne UX |

#### Chatbots généralistes

Pour la réflexion, l'exploration, le brainstorming, la rédaction. Pas limités au code.

| Outil | Éditeur | Spécificités |
|-------|---------|-------------|
| ChatGPT | OpenAI | Écosystème plugins, GPT Store, multimodal |
| Claude | Anthropic | Raisonnement long, fenêtre de contexte large, sécurité |
| Gemini | Google | Intégration Google Workspace, multimodal |
| Mistral (Le Chat) | Mistral AI | Modèles européens, open-weight |

#### Outils spécialisés

Des outils construits pour un cas d'usage précis.

- **v0 (Vercel)** : génération d'interfaces UI à partir d'un prompt
- **Bolt / Lovable** : prototypage d'apps complètes
- **Midjourney / DALL-E** : génération d'images
- **NotebookLM (Google)** : analyse de documents, génération de podcasts
- **Perplexity** : recherche avec citations et sources

### Critères de sélection

Quand vous évaluez un outil IA, cinq critères comptent vraiment.

**1. Qualité de sortie.** Le modèle produit-il du contenu fiable pour votre cas d'usage ? Testez-le sur vos problèmes réels, pas sur des démos.

**2. Confidentialité.** Vos données servent-elles à entrainer le modèle ? Y a-t-il une option "zero data retention" ? Pour du code propriétaire, c'est un critère bloquant.

**3. Intégration.** L'outil s'insère-t-il dans votre workflow existant ? Un outil génial mais qui casse votre flow ne sera pas adopté.

**4. Cout.** La plupart des outils pro coutent entre 20 et 200 dollars par mois. Comparez le cout au temps gagné, pas au prix absolu.

**5. Écosystème.** L'outil a-t-il une API ? Des extensions ? Une communauté active ? Un outil isolé devient vite un cul-de-sac.

### Comparatif synthétique des assistants code

| Critère | Copilot | Cursor | Claude Code |
|---------|---------|--------|-------------|
| Mode d'interaction | Autocomplétion + chat | IDE + chat + agent | CLI agentique |
| Contexte codebase | Fichier ouvert | Multi-fichiers, indexation | Filesystem complet |
| Autonomie | Faible | Moyenne | Élevée |
| Courbe d'apprentissage | Faible | Moyenne | Plus élevée |
| Prix (individuel) | ~10$/mois | ~20$/mois | Basé sur l'usage API |
| Confidentialité | Options entreprise | Options entreprise | Zero retention (API) |

### Ne vous attachez pas à un outil

C'est le conseil le plus important de cette leçon. L'outil que vous utilisez aujourd'hui sera probablement dépassé dans 12 mois. Ce qui reste, ce sont les compétences :

- Savoir formuler un bon prompt
- Comprendre les limites d'un modèle
- Évaluer la qualité d'une sortie
- Intégrer l'IA dans un workflow structuré

Investissez dans ces compétences, pas dans la maitrise d'un outil spécifique. Si vous savez bien utiliser Claude, vous saurez bien utiliser le prochain modèle d'OpenAI ou de Google.

### En résumé

- L'écosystème IA est vaste et évolue vite. Cartographier les catégories vaut mieux que mémoriser les outils.
- Choisissez vos outils selon la qualité, la confidentialité, l'intégration, le cout et l'écosystème.
- Les compétences fondamentales (prompting, évaluation, workflow) sont transférables d'un outil à l'autre.
